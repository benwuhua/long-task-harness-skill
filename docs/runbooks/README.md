# Runbooks

This directory is for diagnosis and recovery procedures that should be available to any worker without oral context.

Add one markdown file per recurring failure mode, dependency outage, or recovery workflow. Good runbooks include:

- trigger signals
- fast triage commands
- diagnosis branches
- recovery steps
- required evidence to capture

Use [`../templates/runbook-template.md`](../templates/runbook-template.md) as the starting point.
